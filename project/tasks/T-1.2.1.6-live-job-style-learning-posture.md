## Description

Two major v1+ features whose *posture* and *novelty status* must be set now:

1. **Live job (A-012).** A job set up before a trip / build / event. App continuously watches a media source (smartphone camera roll, OneDrive folder, Google Photos / iCloud shared upload bucket) and continuously curates. Multi-purpose: a single live job can produce per-location reels + an overall YouTube video + collages per mini-event. Multi-platform: each output can target its own platform. Can publish *during* the event before it ends. Conversationally configured at job creation. Recommendation: phase v1, with a clean feature flag in MVP code so the architecture is ready.

2. **Reference-media style learning (A-014).** User can upload pre-built media, link to internet content, or pick a previous app creation; AI learns its styling, theme, and curation methodology. Recommendation: phase v1.

3. **Patent / novel-idea calls.** Both features are candidate `N-NNN` entries (`N-cand-E` for live-job, `N-cand-D` for reference-media style fingerprint). User decides go/no-go for committing to master per D-005.
