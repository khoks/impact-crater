## Description

Four coupled product decisions:

1. **Job model.** User input (this session): jobs are async — user can leave the app and return; refine-loop is opt-in; publish-approval gate is always on.
2. **Scale target for MVP.** User input: up to 1000 photos + 50 videos per job, with a 2–5 hour wall-clock ceiling end-to-end.
3. **MVP success criterion.** User input: *user drops up to 1000 photos + 50 videos from a single trip/build/event, describes in a paragraph what kind of YouTube video they want and what kind of music, picks a target duration, and gets a publish-ready video to their connected YouTube Studio account within 2–5 hours.*
4. **Effort-level UX with agentic recommendation.** User input: 3–5 effort levels (e.g., L1 = ~10 photos + 1×5min video; L5 = ~10000 photos + 500×20min videos). Based on user's LLM config + quotas + local/remote, the **max permissible** level is computed and surfaced after task details + media selection. Going beyond → cost shown transparently. Going beyond max permissible → app explains the upgrade path in LLM settings. Recommendation, cost-explanation, and upgrade-path UX are agentic + GenAI-generated.
