## Goal

Replace the original "never auto-merge" stance (D-004 / ADR-0003) with an auto-merge-by-default policy for all Claude-generated PRs — `knowledge-curator`, `work-tracker`, and feature PRs alike — using `gh pr merge --squash --delete-branch`. The policy reflects the user's directive (verbatim, 2026-04-26): *"PRs should be automerged be it the work tracker or knowledge curator or actual work PRs."*

This epic captures the documentation, ADR, skill-script, and hook updates needed to make the new behavior the documented default.

## Scope

- File **ADR-0004** documenting the auto-merge decision, alternatives, and consequences.
- Update **ADR-0003** status header to mark the "never auto-merge" clause superseded by ADR-0004; update the relevant decision-list bullet.
- Update the two project-local skills (`work-tracker`, `knowledge-curator`): description frontmatter, intro paragraph, git-flow steps, hard-rules section.
- Update the `Stop` hook script (`.claude/hooks/post-session-housekeeping.sh`) block-reason text to describe the new flow.
- Update `CLAUDE.md`: "Decisions locked" row, "project tracking" + "knowledge curation" sections, "Things to never do" list.
- Append **D-021** to `DECISIONS_LOG.md`.
- Land everything as one PR which itself auto-merges to demonstrate the new flow in action.

## Out of scope

- CI configuration (no CI exists yet; revisit at E-1.3 or first feature work — see ADR-0004 alternatives).
- Branch protection rules on master (acceptable for a single-contributor project at this phase).
- Any change to the branch-and-PR flow itself (it remains in force; only the merge step changed).
