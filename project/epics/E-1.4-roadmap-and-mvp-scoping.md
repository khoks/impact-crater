## Goal

Lock the MVP scope and the v1 / v2 / v3 milestone bundles. By the end of this epic, `MVP.md` is no longer a stub: it names the single critical artifact, the single platform, the single LLM-routing path, the success criterion, and the explicit out-of-scope list. `ROADMAP.md` lists per-phase milestones.

## Scope

- Pick the single MVP artifact type (driven by the answer surfaced in E-1.2).
- Pick the single MVP platform connector.
- Pick the single MVP LLM-routing path (local-first or remote-first).
- Define the MVP success criterion (concrete: "user can take N photos, type a sentence, and post a curated artifact within M minutes").
- File the MVP-scope Initiative (`I-2 MVP`) and break it into Epics for each capability the MVP touches.
- Tag every backlog item across `project/` with the right `phase`, demoting anything beyond MVP to v1 or later.

## Out of scope

- Implementation work — that begins under `I-2 MVP` after this epic closes.
