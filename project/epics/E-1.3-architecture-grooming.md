## Goal

Convert the placeholder `ARCHITECTURE.md` into a real architecture document with accepted ADRs covering the tech stack, the LLM strategy, the media pipeline, the curation engine, the storage layout, the connector layer, the agent harness, and the cross-cutting concerns (resource accounting, privacy, reproducibility, cancellation, telemetry).

## Scope

This is an engineering-led grooming session. Claude proposes options for each architectural decision; the user weighs in (often with innovative or non-obvious ideas of their own); decisions get filed as ADRs in `docs/architecture/` numbered from `ADR-0004` onward, plus matching `D-NNN` entries in `DECISIONS_LOG.md`.

Decisions to make (each is one ADR):

- Process topology and language stack.
- Local-LLM runtime and capability tiers.
- Vision-model lineup at each tier.
- Remote-LLM provider list and abstraction boundary.
- Media pipeline framework (decoders, perceptual hashing, scene detection, smart-crop).
- Curation engine algorithm shape.
- Music alignment strategy.
- Storage layout (filesystem, database, both).
- Connector layer credential model and audit-log shape.
- Agent harness topology (single vs. multi-agent).
- Resource accounting and quota model.
- Privacy posture defaults.

## Out of scope

- The MVP scope itself (E-1.4).
- Any code (writes happen during MVP execution).
