## Goal

Convert the user's verbatim vision in `RAW_VISION.md` into a groomed, phase-tagged feature catalog (`GROOMED_FEATURES.md`) and a candidate-additions list (`RECOMMENDED_ADDITIONS.md`). End the epic with a clear, prioritized list of capabilities the product will ship over time, each tagged with a target phase.

## Scope

This is a product-manager-led grooming session. Claude asks the user crucial product questions; the user supplies the answers. The questions cover (at minimum):

- The single critical artifact the MVP must produce end-to-end.
- The local-first-vs-cloud-first default for media analysis, and the routing rules for hybrid behavior.
- Photo / video selection algorithm shape: deterministic pipeline (perceptual-hash → embedding → cluster → quality score → narrative arc) vs. multimodal-LLM-as-curator vs. hybrid. The user is invited to propose innovative alternatives.
- Music sourcing: licensed library, generated, or user-supplied.
- Inspiration-link learning: official APIs vs. browser-based fallbacks; default off.
- Theme library cold-start strategy.
- Connector strategy: which platforms first; default policy is official APIs only behind an explicit-consent gate.
- Mobile vs. desktop split.
- Agent harness shape: single agent vs. planner + media-analyst + editor multi-agent.

## Out of scope

- Architectural tech-stack choices (those belong to E-1.3).
- Roadmap-phase milestone lists (those belong to E-1.4).
